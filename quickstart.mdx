This guide walks you through installing Claude Code 2.1.88, setting up authentication, and using the interactive REPL to complete a real coding task.

<Note>
  Version 2.1.88 was pulled from the official npm registry. Install it from the Tencent mirror cache — see [Installation](/installation) for full details.
</Note>

## Setup

<Steps>
  <Step title="Install Claude Code">
    Install the CLI globally using the Tencent mirror cache:

    ```bash
    npm install -g https://mirrors.cloud.tencent.com/npm/@anthropic-ai/claude-code/-/claude-code-2.1.88.tgz
    ```

    Verify the installation succeeded:

    ```bash
    claude --version
    ```

    You should see output like `2.1.88 (Claude Code)`.
  </Step>

  <Step title="Set your API key">
    Export your Anthropic API key as an environment variable. Add this to your shell profile (`.bashrc`, `.zshrc`, etc.) to persist it across sessions:

    ```bash
    export ANTHROPIC_API_KEY=sk-ant-api03-...
    ```

    If you prefer OAuth login through your `claude.ai` account instead, skip this step — you will be prompted to log in when you first run `claude`.
  </Step>

  <Step title="Open a project directory">
    Navigate to the codebase you want to work on. Claude Code uses your current directory as the project root:

    ```bash
    cd ~/projects/my-app
    ```
  </Step>

  <Step title="Start the REPL">
    Launch Claude Code:

    ```bash
    claude
    ```

    If you did not set `ANTHROPIC_API_KEY`, Claude will prompt you to log in via OAuth. Follow the URL printed to the terminal to authenticate with your `claude.ai` account.

    Once authenticated, you will see the interactive prompt ready to accept your first request.
  </Step>

  <Step title="Run your first task">
    Type a natural-language request and press Enter:

    ```
    > Add error handling to my Express routes so unhandled errors return a JSON response with status 500
    ```

    Claude will read the relevant files, propose a plan, ask for your approval, apply the edits, and optionally run your test suite to verify nothing broke.
  </Step>
</Steps>

## Example session

Here is a realistic end-to-end session showing how Claude handles a multi-file refactor:

```
> Refactor the user authentication middleware in src/middleware/auth.ts to use
  async/await instead of callbacks, and update the tests in tests/auth.test.ts

● Reading src/middleware/auth.ts...
● Reading tests/auth.test.ts...

I'll refactor the auth middleware and update the tests. Here's my plan:

1. Convert the callback-based verifyToken function to async/await
2. Update the Express middleware wrapper to be an async function
3. Update the two test cases that mock the callback interface

May I proceed with these 3 file edits? [y/n] y

● Editing src/middleware/auth.ts...  ✓
● Editing tests/auth.test.ts...      ✓
● Running: npm test                  ✓ 12 passed

Done. The middleware and tests have been updated.
```

## Slash commands

Inside the REPL, slash commands give you direct access to Claude Code's subsystems. Type `/` followed by the command name:

| Command | What it does |
|---|---|
| `/help` | Show all available commands and keyboard shortcuts |
| `/config` | View and edit configuration (model, permissions, memory settings) |
| `/mcp` | Add, remove, enable, or disable MCP servers |
| `/memory` | Open the project or user memory file — Claude reads these at session start |
| `/permissions` | Review and modify tool permission rules |
| `/review` | Start a structured code review of staged changes or a file range |
| `/session` | List, resume, or export previous conversations |
| `/tasks` | View and manage the active task graph for multi-step work |
| `/plan` | Enter planning mode — Claude outlines a full plan before taking action |
| `/compact` | Summarize the conversation to free up context window space |
| `/model` | Switch the active model mid-session |
| `/cost` | Show token usage and estimated cost of the current session |
| `/exit` | Exit the REPL |

<Tip>
  Press `Ctrl+C` once to cancel an in-progress response. Press `Ctrl+C` twice (or type `/exit`) to quit the REPL.
</Tip>
