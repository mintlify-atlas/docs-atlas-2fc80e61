title: "Commands overview"
description: "Complete reference for all slash commands available in the Claude Code REPL. Type / to open the command picker, or type a command directly."
---

Slash commands give you direct control over Claude Code without leaving the terminal. Type `/` in the REPL to open the interactive picker with fuzzy search, or type a command directly (e.g. `/compact`).

<Note>
Commands are loaded from four sources: built-in commands, installed plugins, skills directories (`.claude/skills/`), and MCP servers. The list you see may vary depending on which plugins and MCP servers you have enabled.
</Note>

## Session & navigation

| Command | Description | Example |
|---------|-------------|---------|
| `/clear` | Clear conversation history and free up context. Aliases: `reset`, `new` | `/clear` |
| `/exit` | Exit the REPL. Alias: `quit` | `/exit` |
| `/session` | Show remote session URL and QR code (requires `--remote` mode). Alias: `remote` | `/session` |
| `/resume` | Resume a previous conversation by ID or search term. Alias: `continue` | `/resume` or `/resume my-feature-branch` |
| `/rewind` | Restore code and/or conversation to a previous checkpoint. Alias: `checkpoint` | `/rewind` |
| `/compact` | Summarize and compress conversation history to free up context window tokens | `/compact` or `/compact focus on the auth module` |
| `/export` | Export the current conversation to a file or clipboard | `/export` or `/export session.md` |

## AI & code quality

| Command | Description | Example |
|---------|-------------|---------|
| `/review` | AI code review of a pull request. Reads `gh pr diff` and analyzes changes | `/review` or `/review 42` |
| `/security-review` | Security-focused code review of your current changes | `/security-review` |
| `/plan` | Enable plan mode or view the current session plan. When enabled, Claude presents a step-by-step plan and waits for your approval before executing | `/plan` or `/plan open` |
| `/ultrareview` | Deep bug-finding review that runs in the cloud (~10–20 min). Finds and verifies bugs in your branch | `/ultrareview` or `/ultrareview 42` |
| `/diff` | View uncommitted changes and per-turn diffs | `/diff` |

## Command types

Commands fall into three implementation types that determine how they behave:

<AccordionGroup>
  <Accordion title="local">
    Runs synchronously in the REPL process and returns text output. Examples: `/clear`, `/compact`, `/vim`. These are the lightest commands and often support non-interactive (SDK) mode.
  </Accordion>
  <Accordion title="local-jsx">
    Renders a full React/Ink UI panel in the terminal. Examples: `/config`, `/mcp`, `/model`. These commands display interactive pickers, forms, and panels. They do not run in non-interactive mode.
  </Accordion>
  <Accordion title="prompt">
    Expands to a prompt that is sent to the Claude model. Examples: `/review`, `/security-review`, `/pr-comments`. The model then executes the task using available tools. Skills loaded from `.claude/skills/` are also prompt-type commands.
  </Accordion>
</AccordionGroup>
